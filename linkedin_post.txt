🚀 Zero dependencies. 100% secure. Deployed. Built in public.

I have engineered and deployed EcoTrace—an advanced, secure, and fully accessible Carbon Footprint Dashboard for the Carbon Footprint Awareness Platform mini-challenge. 🌍

Here is a summary of the architectural and technical decisions behind this bare-metal build:

🛠️ The Tech Stack & Architecture

Frontend: Semantic HTML5, Vanilla CSS3, and Modular ES6+ JavaScript.

Data Visualization: Built using raw SVG elements and trigonometric arcs, avoiding heavy libraries like Chart.js or D3.

Server & Deployment: Native Node.js HTTP and File System modules (zero package.json dependencies), deployed to Vercel Serverless CDNs using custom edge security configurations (vercel.json).

Testing: Node.js native test runner (node:test and node:assert) for decimal and boundary validations, achieving a 100% test pass rate.

🔒 Production-Grade Security & Compliance

Security Hardening: Implemented directory traversal mitigation alongside strict HTTP headers, including a robust Content Security Policy (CSP), X-Frame-Options: DENY, and X-Content-Type-Options: nosniff.

WCAG 2.1 AA Accessibility: Integrated full WAI-ARIA keyboard navigation across dashboard tabs, dynamic error binding (aria-invalid/aria-describedby), and polite live regions (aria-live) for real-time assistive technology announcements.

🤖 Offline-First Intelligence

Developed EcoBot, a client-side assistant that parses user carbon habits locally to compute targets and outline transport savings without external API overhead.

Building this from the ground up offered an excellent opportunity to balance strict security compliance, optimal performance, and inclusive design using native web APIs.
